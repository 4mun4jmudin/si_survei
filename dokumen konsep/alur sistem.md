## 2) Alur sistem

Alur yang saya sarankan:

### Alur utama

1. Admin login.
2. Admin membuat periode survei.
3. Admin input pertanyaan survei.
4. Siswa login dan mengisi survei.
5. Jawaban disimpan ke database.
6. Sistem menghitung hasil per pertanyaan, per indikator, dan per responden.
7. Sistem menjalankan pengelompokan hasil.
8. Admin membuka laporan dan grafik.
9. Laporan bisa dicetak atau diekspor.

### Alur responden

1. Siswa login.
2. Sistem cek apakah siswa sudah pernah isi survei pada periode itu.
3. Jika belum, tampilkan form survei.
4. Siswa menjawab.
5. Simpan jawaban.
6. Tampilkan status selesai.

### Alur admin

1. Login.
2. Kelola pertanyaan.
3. Kelola periode survei.
4. Lihat hasil.
5. Cetak laporan.

Ini nyambung dengan bagian laporan yang menyebut sistem akan menyajikan hasil survei dalam bentuk grafik, tabel, dan laporan evaluasi, serta digunakan untuk evaluasi rutin lingkungan belajar.
